Esta página lista las *features* que el framework Glory expone actualmente para ser habilitadas
o deshabilitadas desde código (ej. `App/Config/control.php`) usando `GloryFeatures::enable()` / `disable()`
o bien desde el panel cuando exista la opción asociada.

Por convención estas se agrupan por responsabilidad (Managers, UI, Services, Renderers, Theme options).

## Cómo controlar una feature

```php
use Glory\Core\GloryFeatures;

// Habilitar
GloryFeatures::enable('nombreFeature');

// Deshabilitar
GloryFeatures::disable('nombreFeature');
```

También puedes usar `GloryFeatures::isActive('nombreFeature')` para condicionar registro de assets o hooks.

## Notas

- La lista anterior se obtuvo de `App/Config/control.php`. Mantén este archivo sincronizado con la lista de features
  en la documentación cuando añadas o elimines features.
- Para features que requieran assets, recuerda usar la clave `feature` al definir assets en `AssetManager` para
  que el propio manager respete el estado de la feature automáticamente.
