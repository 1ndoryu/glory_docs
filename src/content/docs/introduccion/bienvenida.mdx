Glory es un conjunto de clases y utilidades para construir temas en WordPress con un enfoque claro: mantener el núcleo agnóstico y mover la personalización al tema. Proporciona managers y componentes reutilizables para organizar mejor el código y reducir repetición.

## Cuándo usarlo

- Cuando necesitas declarar y sincronizar páginas desde código (en lugar de crearlas manualmente).
- Cuando quieres gestionar assets del tema de forma consistente y versionada.
- Cuando prefieres separar responsabilidades: reutilizable en `Glory/`, específico del proyecto en `App/`.

## Principios

- Glory (núcleo) es agnóstico: no contiene lógica de negocio del proyecto.
- El tema (`App/`) concentra las decisiones concretas del sitio.
- Managers y componentes ayudan a escribir menos código repetitivo y más mantenible.

## Arquitectura (alto nivel)

- `Glory/`: clases reutilizables (managers, utilidades, componentes) sin atar a un proyecto concreto.
- `App/`: funciones/configuración del tema. Aquí defines tus páginas, assets y opciones del proyecto.

En particular:
- `AssetManager`: registra y encola CSS/JS del tema.
- `PageManager`: define páginas por `slug` y sincroniza su existencia y ajustes.

Consulta las páginas dedicadas para detalles y API:
- [AssetManager](/managers/asset-manager)
- [PageManager](/managers/page-manager)

## Ejemplo mínimo (orientativo)

```php
<?php
use Glory\Manager\PageManager;

// Definir y sincronizar una página básica
PageManager::define('mi-pagina', 'mi_pagina_render');
PageManager::register();
PageManager::procesarPaginasDefinidas();
```

Nota: la configuración concreta del tema (dónde colocar estas llamadas) depende de tu proyecto. La guía completa está en [PageManager](/managers/page-manager).

## Modo Dev/Local (breve)

- En entornos de desarrollo, algunas tareas se facilitan (p. ej., sincronización automática de páginas y comportamiento de caché para assets). Revisa los apartados «Modo Dev/Local» en las páginas de cada manager para detalles y requisitos.

## Errores frecuentes

- Colocar lógica específica del proyecto dentro de `Glory/` (debe ir en `App/`).
- Documentar o depender de features no verificadas en el código real.
- Duplicar handles de assets o usar rutas incorrectas al registrarlos.
