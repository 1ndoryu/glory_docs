## Resumen

Declara páginas por `slug` desde código y deja que `TemplateGlory.php` ejecute tu función de render cuando aplique. En DEV, la sincronización ocurre automáticamente.

## Cuándo usarlo

- Quieres que ciertas páginas existan siempre con ajustes controlados por código.
- Prefieres handlers por función centralizados en lugar de múltiples plantillas sueltas.
- Necesitas reproducibilidad entre entornos (sin depender de crear páginas a mano).

## API básica (uso común)

```php
<?php
use Glory\Manager\PageManager;

// Handler por función (usa TemplateGlory.php para render)
PageManager::define('contacto', 'contactoRender');

// Página basada en plantilla .php específica (sin handler por función)
PageManager::define('sobre-nosotros', 'TemplateSobreNosotros.php');
```

Parámetros:
- `slug`: `kebab-case`, único por página.
- `handler`:
  - Si termina en `.php`, se usa como plantilla.
  - Si es nombre de función, se usará `TemplateGlory.php` y se llamará a esa función.
- `roles` (opcional, cuarto parámetro): fuerza login y valida roles.

La API completa y casos avanzados están en [PageManager](/managers/page-manager).

## Ejemplo mínimo

1) Define una página en `App/Config/pages.php`:

```php
<?php
use Glory\Manager\PageManager;

PageManager::define('demo', 'demo');
```

2) Crea el archivo de la página en `App/Templates/pages/demo.php`:

```php
<?php

function demo()
{
?>
    <section class="hero">
        <h1>Bienvenido</h1>
        <p>Esta es una página de ejemplo.</p>
    </section>

    <section class="contenido">
        <h2>Sección de contenido</h2>
        <p>Aquí va el contenido de tu página.</p>
    </section>
<?php
}
```

**Importante:**
- No uses `<main>` en tu función, ya lo incluye el header del tema.
- No uses `echo` para el HTML, abre y cierra PHP para escribir HTML directamente.
- Los estilos de la página deben ir en `App/Assets/css/`, no inline en la función.

Notas:
- `TemplateGlory.php` (incluido en el tema) llama a `PageManager::getFuncionParaRenderizar()` y ejecuta la función.
- Si prefieres una plantilla dedicada, usa `PageManager::define('demo', 'TemplateDemo.php');`.

## Integración con el tema

- Ubicación de definiciones: `App/Config/pages.php`.
- Ubicación de páginas: `App/Templates/pages/`.
- Estilos de páginas: `App/Assets/css/`.
- Evita lógica de negocio en `Glory/` (núcleo agnóstico); mantenla en `App/`.

## Modo Dev/Local

- `.env` admite `LOCAL=true` y `DEV=true` (ver `App/Config/environment.php`).
- En DEV:
  - Admin: sincronización completa (opciones, páginas, contenido por defecto).
  - Frontend: sincronización ligera de páginas gestionadas.
- En producción (`DEV=false`): usa la barra «Glory Sync» para sincronizar manualmente.

## Errores frecuentes

- Slug inválido (usa sólo minúsculas, números y guiones).
- Falta `TemplateGlory.php` cuando defines handler por función.
- Handler inexistente o no cargado por el tema.
- Esperar sincronización automática en producción (usa «Glory Sync»).
