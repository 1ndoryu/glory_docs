## Resumen

Usa componentes del núcleo (`Glory/`) desde tu tema (`App/`) para evitar repetir lógica/HTML. Activa la feature del componente y llama su API mínima.

## Cuándo usarlo

- Quieres reutilizar UI o renderizados comunes sin duplicar código.
- Buscas consistencia y rapidez (listas de posts, logo del tema, modales básicos).
- Necesitas mantener el núcleo agnóstico y la personalización en el tema.

## Ejemplos mínimos

### 1) Logo del tema (LogoRenderer)

Asegúrate de tener activa la feature en `App/Config/control.php`:

```php
Glory\Core\GloryFeatures::enable('logoRenderer');
```

Uso vía shortcode (rápido):

```php
<?php
echo do_shortcode('[theme_logo width="120px" filter="white"]');
```

Uso en PHP (control total):

```php
<?php
use Glory\Components\LogoRenderer;

echo LogoRenderer::get_html([
    'width'  => '120px',      // opcional
    'filter' => 'white',      // '', 'white', 'black' o cualquier CSS filter
]);
```

Más detalles: [LogoRenderer](/componentes/renderers/logo-renderer)

### 2) Listas de contenido (ContentRender)

Renderiza posts de forma declarativa (con caché en producción):

```php
<?php
use Glory\Components\ContentRender;

ContentRender::print('post', [
    'publicacionesPorPagina' => 6,
    // 'paginacion' => true,
    // 'plantillaCallback' => [MiClase::class, 'miPlantilla'],
]);
```

- En DEV (WP_DEBUG true) no se usa caché.
- Puedes personalizar la plantilla pasando `plantillaCallback($post, $itemClass)`.

Más detalles: [ContentRender](/componentes/renderers/content-render)

### 3) Modal básico (Modal)

Activa la feature si tu proyecto usa estilos/JS asociados (p. ej. `modales`):

```php
Glory\Core\GloryFeatures::enable('modales');
```

Render mínimo en PHP:

```php
<?php
use Glory\Components\Modal;

echo Modal::render('miModal', 'Título', '<p>Contenido del modal</p>');
```

Luego, muestra/oculta el modal con tu JS preferido (por ejemplo, togglear `display`).

## Integración con el tema

- Activa sólo las features que uses en `App/Config/control.php`.
- Si un componente requiere CSS/JS del tema, define assets en `App/Config/assets.php` con `AssetManager`.
- Mantén la lógica de negocio y personalizaciones en `App/` (el núcleo `Glory/` es agnóstico).

## Modo Dev/Local

- En DEV (`DEV=true` en `.env`): assets auto‑descubiertos sin caché; sincronizaciones automáticas activas según el manager.
- En producción (`DEV=false`): usa la barra «Glory Sync» para sincronizar; controla versión de assets con `AssetManager::setThemeVersion(...)`.

## Errores frecuentes

- Olvidar activar la feature del componente en `control.php`.
- Suponer métodos que no existen (usa `get_html(...)` en `LogoRenderer`, no instancias).
- Rutas de assets del tema mal puestas (usa `App/Assets/...`).
- Esperar comportamiento de DEV en producción (caché y minificados cambian).
